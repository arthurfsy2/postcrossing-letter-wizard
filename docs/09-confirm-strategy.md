# 每步确认策略（Confirmation Strategy）

> 📋 **对应 SKILL.md 章节**：执行流程 → 每步确认  
> **优先级**：中  
> **完成时间**：2026-03-26

---

## 功能说明

确保用户在每个关键步骤都能确认或纠正，避免全自动导致的失控感。

---

## 策略

| 场景 | 策略 |
|------|------|
| **首次使用**（无 `user-content-template.md`） | 每步确认，等待用户反馈 |
| **后续使用**（有模板文件） | 询问"使用上次配置？"→ 是则全自动 |

---

## 确认点设计

### Step 1: 配置完成

```
✅ 邮箱配置已保存：xxx@qq.com (国家：CN, 城市：Shenzhen)

请确认配置无误：
- 继续 → 进入 Step 2：搜索邮件
- 修改 → 返回修改配置（邮箱/城市/国家）
```

### Step 2.5: 原始邮件保存完成

```
📧 已保存 10 封原始邮件到 2026-01-29/2026-01-29_raw-emails.md

查看文件确认内容无误后，请回复：
- 继续 → 进入 Step 3：分析收件人
- 修改 → 重新选择邮件或日期范围
```

### Step 3: 分析完成

```
📊 已分析 10 位收件人，保存到 2026-01-29/2026-01-29_recipient-analysis.md

查看文件确认内容无误后，请回复：
- 继续 → 进入 Step 4：收集用户素材
- 修改 → 重新分析或调整偏好
```

### Step 4: 素材收集完成

```
📝 个人素材已保存到 user-content-template.md

确认素材足够后，请回复：
- 继续 → 进入 Step 5：生成明信片内容
- 补充 → 继续添加更多个人素材
```

### Step 5: 生成完成

```
✉️ 明信片内容已生成到 2026-01-29/2026-01-29_postcard-content.md

请选择输出格式：
- A. Markdown 文件 → 默认格式，可复制手写
- B. HTML 打印版 → A4 批量打印，含剪切线
- C. 两者都要 → 同时生成两种格式

请回复 A、B 或 C：
```

### Step 6: 邮件发送完成（可选）

```
✅ 邮件已发送到 xxx@qq.com

压缩包包含：
- 2026-01-29_raw-emails.md
- 2026-01-29_recipient-analysis.md
- 2026-01-29_postcard-content.md

流程已全部完成！🎉
```

---

## 实现要点

1. **每个步骤完成后输出明确的确认提示**
2. **用户回复"继续/是/OK"才进入下一步**
3. **用户回复"修改/不对"则返回上一步修正**
4. **首次使用引导用户逐步确认，建立信任**

---

## 用户响应处理

| 用户回复 | 处理方式 |
|----------|----------|
| "继续"、"是"、"OK"、"好的" | 进入下一步 |
| "修改"、"不对"、"返回"、"重新" | 返回上一步修正 |
| "取消"、"停止"、"结束" | 流程结束，提醒用户可随时重新触发 |
